# Анализ клиентского поведения, удержания и RFM-сегментации в e-commerce

Комплексный аналитический проект на данных интернет-магазина с использованием **Python**, **SQL Server** и **Power BI**.

## О проекте

Цель проекта — проанализировать поведение клиентов, оценить удержание после первой покупки и выделить клиентские сегменты по ценности с помощью **RFM-анализа**.

В рамках проекта были выполнены:
- очистка и исследовательский анализ данных в Python
- построение аналитических витрин в SQL Server
- когортный анализ удержания клиентов
- RFM-сегментация
- визуализация результатов в Power BI

## Бизнес-цель

Понять:
- как меняются выручка, клиентская база и покупательская активность во времени
- как удерживаются клиенты после первой покупки
- какие сегменты клиентов приносят наибольшую ценность
- какие сегменты требуют удержания или реактивации

## Датасет

Датасет содержит транзакции интернет-магазина с информацией о клиентах, товарах, дате покупки, способе оплаты и базовых демографических признаках.

Основные поля, использованные в анализе:
- `customer_id`
- `purchase_date`
- `product_category`
- `payment_method`
- `quantity`
- `product_price`
- `gender`
- `age`

## Этапы работы
### 1. Подготовка данных и EDA в Python

На этом этапе были:

* проверены пропуски, дубликаты и аномальные значения
* удалены лишние столбцы
* исследована динамика выручки, числа клиентов и покупательских событий по месяцам
* рассчитаны интервалы между покупками
* зафиксированы правила расчета повторных покупок и customer inactivity
### 2. Построение витрин в SQL Server

Были собраны следующие аналитические слои:

* `fact_transactions` — транзакционная таблица с расчетом выручки
* `dm_monthly_kpi` — месячные KPI: выручка, число клиентов, число событий
* `dm_customer_features` — признаки на уровне клиента: recency, frequency, revenue, repeat flag
* `dm_cohort_retention` — когортная таблица удержания
* `dm_rfm_segments` — RFM-оценки, коды и клиентские сегменты
### 3. Визуализация в Power BI

В Power BI были собраны:

* графики месячной динамики
* cohort retention heatmap
* дашборд RFM-сегментации
## Логика расчета ключевых метрик
### Выручка

Выручка рассчитывалась по формуле:

`revenue = product_price * quantity`

### Cohort Retention
* `cohort_month` — месяц первой покупки клиента
* `cohort_index` — количество месяцев с момента первой покупки
* `retention_rate` — доля активных клиентов относительно размера когорты
## RFM-анализ

### RFM-сегментация строилась на основе трех показателей:

* Recency — сколько дней прошло с последней покупки клиента
* Frequency — как часто клиент совершал покупки
* Monetary — какую суммарную выручку принес клиент

После этого клиенты были распределены по сегментам:

* Champions
* Loyal
* Potential
* Big Spenders
* At Risk
* Lost
## Основные результаты
* Динамика выручки тесно связана с числом активных клиентов и числом покупательских событий.
* Когортный анализ показал ожидаемое снижение retention после первого месяца покупки.
* Клиентская база неоднородна по ценности и активности.
* RFM-сегментация показала, что наиболее ценные сегменты — Champions и At Risk, тогда как Lost — один из самых крупных сегментов по числу клиентов, но с низкой средней ценностью.
* По итоговому дашборду было проанализировано 18.39K клиентов и 38.2M общей выручки.
## Бизнес-интерпретация

Полученные сегменты позволяют по-разному работать с клиентской базой:

* **Champions** — удерживать и дополнительно поощрять
* **Loyal** — поддерживать вовлеченность
* **Potential** — развивать в более ценные сегменты
* **Big Spenders** — повышать частоту покупок
* **At Risk** — реактивировать
* **Lost** — оценивать целесообразность возвратных кампаний
